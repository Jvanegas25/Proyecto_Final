API Gestión de Pedidos

Descripción: API REST para gestionar clientes y pedidos.

Base URL: http://127.0.0.1:8000/api/

Formato de respuesta: JSON

Framework: Django REST Framework

Base de datos: PostgreSQL

Lista de Endpoints

Método URL Descripción
GET /api/clientes/ Listar todos los clientes
POST /api/clientes/ Crear un nuevo cliente
GET /api/clientes/{id}/ Obtener un cliente por ID
PUT /api/clientes/{id}/ Actualizar un cliente
DELETE /api/clientes/{id}/ Eliminar un cliente
GET /api/pedidos/ Listar todos los pedidos
POST /api/pedidos/ Crear un nuevo pedido
GET /api/pedidos/{id}/ Obtener un pedido por ID
PUT /api/pedidos/{id}/ Actualizar un pedido
DELETE /api/pedidos/{id}/ Eliminar un pedido

Clientes

GET /api/clientes/

Devuelve la lista de todos los clientes registrados.

Ejemplo de respuesta:

[
  {
    "id": 1,
    "nombre": "Juan Pérez",
    "email": "juan@email.com",
    "telefono": "8888-1234"
  },
  {
    "id": 2,
    "nombre": "María López",
    "email": "maria@email.com",
    "telefono": "7777-5678"
  }
]

POST /api/clientes/

Crea un nuevo cliente. Se deben enviar los datos en el cuerpo de la solicitud.

Ejemplo de request:

{
  "nombre": "Carlos Jiménez",
  "email": "carlos@email.com",
  "telefono": "6666-4321"
}

Ejemplo de respuesta (201 Created):

{
  "id": 3,
  "nombre": "Carlos Jiménez",
  "email": "carlos@email.com",
  "telefono": "6666-4321"
}

GET /api/clientes/{id}/

Devuelve los datos de un cliente específico según su ID.

Parámetro: id — ID del cliente (número entero)

Ejemplo de respuesta:

{
  "id": 1,
  "nombre": "Juan Pérez",
  "email": "juan@email.com",
  "telefono": "8888-1234"
}

PUT /api/clientes/{id}/

Actualiza los datos de un cliente existente.

Parámetro: id — ID del cliente a actualizar

Ejemplo de request:

{
  "nombre": "Juan Pérez Actualizado",
  "email": "juan_nuevo@email.com",
  "telefono": "5555-0000"
}

Ejemplo de respuesta (200 OK):

{
  "id": 1,
  "nombre": "Juan Pérez Actualizado",
  "email": "juan_nuevo@email.com",
  "telefono": "5555-0000"
}

DELETE /api/clientes/{id}/

Elimina un cliente según su ID.

Parámetro: id — ID del cliente a eliminar

Respuesta: 204 No Content (sin cuerpo de respuesta)

Pedidos

GET /api/pedidos/

Devuelve la lista de todos los pedidos registrados.

Ejemplo de respuesta:

[
  {
    "id": 1,
    "cliente": 1,
    "producto": "Laptop Dell",
    "cantidad": 2,
    "estado": "pendiente",
    "fecha": "2026-04-19T10:00:00Z"
  }
]

POST /api/pedidos/

Crea un nuevo pedido asociado a un cliente existente.

Ejemplo de request:

{
  "cliente": 1,
  "producto": "Monitor LG 27",
  "cantidad": 1,
  "estado": "pendiente"
}

Ejemplo de respuesta (201 Created):

{
  "id": 2,
  "cliente": 1,
  "producto": "Monitor LG 27",
  "cantidad": 1,
  "estado": "pendiente",
  "fecha": "2026-04-19T10:05:00Z"
}

GET /api/pedidos/{id}/

Devuelve los datos de un pedido específico según su ID.

Parámetro: id — ID del pedido

Ejemplo de respuesta:

{
  "id": 1,
  "cliente": 1,
  "producto": "Laptop Dell",
  "cantidad": 2,
  "estado": "pendiente",
  "fecha": "2026-04-19T10:00:00Z"
}

PUT /api/pedidos/{id}/

Actualiza los datos de un pedido, por ejemplo cambiar el estado a "entregado".

Parámetro: id — ID del pedido a actualizar

Ejemplo de request:

{
  "cliente": 1,
  "producto": "Laptop Dell",
  "cantidad": 2,
  "estado": "entregado"
}

Ejemplo de respuesta (200 OK):

{
  "id": 1,
  "cliente": 1,
  "producto": "Laptop Dell",
  "cantidad": 2,
  "estado": "entregado",
  "fecha": "2026-04-19T10:00:00Z"
}

DELETE /api/pedidos/{id}/

Elimina un pedido según su ID.

Parámetro: id — ID del pedido a eliminar

Respuesta: 204 No Content (sin cuerpo de respuesta)
